Oracle Cloud Trial 신청: Universal Credit

October 29, 2017 oracle cloud

Oracle은 무료로 Oracle Cloud를 체험할 수 있는 Oracle Cloud Trial 프로그램을 운영하고 있습니다. Oracle Cloud Trial 신청 방법이 2017년 10월 1일 Oracle Universal Credit Pricing 방식이 새로 도입되면서 변경되었습니다. 이 새로운 계정 신청 방식은 2017년 10월 28일부터 한국에는 적용되었습니다. 새롭게 변경된 Oracle Cloud Trial 계정 신청 방법을 정리합니다.

Oracle Trial 계정을 신청하면 “$300 Credit“을 제공합니다. $300 Credit이란 Oracle Cloud 사용 권한으로, Oracle Cloud를 $300 Credit으로 자유롭게 사용할 수 있습니다. Oracle Cloud 서비스를 사용하면 지급한 Credit은 분 단위로 차감됩니다. 차감되는 비용은 서비스별로 차이가 있습니다. 각 서비스 과금 비용은 Oracle Cloud 홈페이지(http://cloud.oracle.com )에서 확인할 수 있습니다. $300 Credit의 유효 기간은 신청일로부터 30일입니다. Oracle Cloud Trial 프로그램에서 다음과 같은 서비스를 체험해 볼 수 있습니다. (2017년 10월 29일 기준 목록)

  • Compute(Bare Metal)
  • Compute Classic
  • Storage Classic
  • Java
  • Database Classic
  • Analytics

  • Container Classic
  • Data Integration Platform Cloud
  • Database (Bare Metal)

  • SOA
  • Visual Builder

Oracle Cloud Trial 등록 준비사항

Oracle Cloud Trial을 신청하기 위해서는 E-mail 계정, 핸드폰 번호, 신용카드가 필요합니다. E-mail은 회사 메일과 개인 메일 모두 가능합니다. 예전에는 Oracle Cloud Trial 신청 시 회사 메일을 사용할 것을 권장했지만, 현재 Oracle Cloud Trial 프로그램은 회사 메일과 개인 메일을 동등하게 취급합니다.

개인 메일을 사용할 경우 “Naver“나 “Daum” 메일 사용을 권장합니다. Gmail을 사용할 경우 Oracle Cloud Trial 승인이 오래 결리는 현상이 발생하고 있습니다. 따라서 원활한 진행을 위해서 Gmail이 아닌 “Naver” 혹은 “Daum” 메일로 오라클 계정을 생성하는 것을 권장합니다.

핸드폰 번호는 SMS 인증 용도로 사용됩니다. Oracle Cloud Trial 신청에 핸드폰 번호를 재사용할 수 없습니다. Oracle Cloud Trial 신청에 사용된 핸드폰 번호로 다른 Oracle Cloud Trial을 신청할 수 없습니다.

신용카드는 사용자 인증 용도로 1회 사용됩니다. 사용자 인증 용도로 Oracle Cloud Trial 등록 시점에 $1을 임시 결제 합니다. $1 결제는 실제로 청구되는 금액이 아니며, 사용자 인증 수단으로만 사용됩니다.

Oracle Cloud 트라이얼 계정 신청은 http://cloud.oracle.com 에서 진행됩니다. http://cloud.oracle.com 은 Oracle Cloud의 홈페이지 입니다. <그림 1 참조>

<그림 2>와 같이 “한국어”를 선택하여, 홈페이지 언어를 한글로 전환합니다.

<그림 3>과 같이 Oracle Cloud 홈페이지에서 “무료 체험” 버튼을 클릭하여 Oracle Cloud Trial 신청 페이지로 이동합니다.

<그림 4>과 같이 “세부정보 보기 및 지금 등록” 버튼을 클릭하여 Oracle Cloud Trial 등록 페이지로 이동합니다.

Oracle Cloud Trial 등록 페이지는 <그림 5>와 같이 4개 부분으로 구성되어 있습니다. 이 페이지에서 모든 트라이얼 계정 등록이 완료됩니다.

인증 코드 등록

트라이얼 계정 등록 두 번째는 “인증 코드“입니다. 핸드폰 전화번호를 등록하고, SMS를 이용한 인증 코드 검증 과정이 진행됩니다. <그림 8 참조> 인증이 완료되지 않으면, 다음 단계로 진행할 수 없습니다.

<그림 8>과 같이 핸드폰 번호를 입력하고 “코드 전송” 버튼을 클릭합니다.

<그림 8>에서 입력한 전화번호로 <그림 9>와 같은 SMS가 전송됩니다.

메시지에 포함된 7자리 숫자를 <그림 10>에 입력하고 “확인” 버튼을 클릭하면 “지급 방법 추가” 버튼이 활성화되는 것을 확인할 수 있습니다.

지급 방법 등록

“SMS 인증”을 완료하면 <그림 11>과 같이 “지급 방법 추가” 버튼을 클릭하여 신용 카드 등록을 진행합니다.

신용카드 등록에 앞서, 계정 세부 정보 등록에서 입력한 주소를 검증합니다.

<그림 13>과 같이 입력한 주소가 출력됩니다. 아래로 스크롤 하면 신용카드 입력 폼이 나타납니다. Visa와 Master 카드를 정보를 입력하고 “마침” 버튼을 클릭하여 신용카드를 추가합니다.

<그림 14>에서 등록한 신용카드에는 사용자 검증 용도이며 별도의 과금이 발생하지 않습니다. Oracle Cloud Trial 등록 시 $1이 사용자 확인 용도로 가결제 됩니다. 사용자 확인 즉시 가결제는 취소되며, 실제 카드 청구에 반영되지 않습니다.

신용 카드 등록이 정상적으로 완료되면 <그림 15>와 같이 카드 번호가 출력됩니다.

조항 및 조건

이제 트라이얼 계정 등록 미지막 단계가 남았습니다. <그림 16>과 같이 트라이얼 계정 라이센스에 동의한 후, “완료” 버튼을 클릭하면 모든 등록 절차는 완료됩니다.

모든 등록이 완료되면 <그림 17>과 같이 등록 완료 페이지가 출력됩니다. <그림 17>에서 계정 활성화에 15분 정도 소요된다는 메시지를 확인할 수 있습니다.

계정 생성 완료 - 메일 발송

대부분의 경우 15분 정도면 Oracle Cloud 트라리얼 계정 생성이 완료됩니다. 계정 활성화가 15분 이상 걸릴 경우 <그림 18>과 같은 안내 메일이 발송됩니다. <그림 18>의 안내 메일은 계정 활성화에 시간이 약간 더 필요하다는 안내 메일 입니다. 개인 메일(다음/네이버 등)을 사용할 경우, Oracle Cloud 발송 메일은 스팸 메일로 분류될 수 있습니다. <그림 18> 역시 스팸 메일로 분류된 상태입니다. 데모에서는 약 17분 정도 소요되었습니다.

Oracle Cloud 트라이얼 계정 생성이 완료되면 <그림 19>와 같은 메일이 발송됩니다. 계정 생성 완료 메일에는 트라이얼 계정 사용자 아이디, 로그인 패스워드 및 클라우드 계정 정보가 포함되어 있습니다. Oracle Cloud 발송 메일은 스팸 메일로 분류될 수 있습니다

Oracle Cloud 트라이얼 계정 로그인

<그림 19> 메일에서 계정 정보를 확인한 후, “http://cloud.oracle.com"으로 이동합니다. <그림 20>과 같이 “Sign In” 메뉴를 클릭하고 로그인합니다.

Oracle Cloud에 로그인하기 위해서 <그림 21>과 같이 “Public Cloud Service - US”를 선택합니다. Oracle Cloud 트라이얼 계정 생성 과정에서 데이터 센터를 “North America”를 선택하면, “Public Cloud Service - US”에 계정이 만들어집니다. 데이터센터 선택 후 My Services 버튼을 클릭합니다.

<그림 22>와 같이 Oracle Cloud에 로그인하기 위해서는 Identity Domain을 입력해야 합니다. 이 정보는 <그림 19> 메일에서 확인할 수 있습니다.

<그림 23>에서는 최종 확인 메일에 포함된 id/password 정보를 입력합니다.

시스템이 생성한 임시 패스워드로 로그인하면, 패스워드를 변경을 강제합니다. 계정 패스워드 분실 시 사용할 질문을 선택하고 답을 등록합니다. <그림 24 참조> 패스워드 설정과 질문 등록이 완료되면 “제출“버튼을 클릭합니다.

Oracle Cloud 대시보드

패스워드 재설정 페이지가 완료되면, <그림 25>와 같은 대시보드가 출력됩니다. 현재 대시보드에는 “SDG400” credit 메시지를 확인할 수 있습니다. SDG는 싱가폴 달러를 의미합니다.

대시보드에서 접근 가능한 서비스를 확인하기 위해서는 <그림 26>과 같이 “대시보드 사용자 정의”를 클릭해야 합니다.

<그림 26>에서 “대시보드 사용자 정의”를 클릭하면 <그림 27>과 같이 접근 가능한 서비스 목록이 출력됩니다.

이 중에서 “Compute Classic”을 “표시”로 설정하면 <그림 28>과 같이 대시보드에 “Compute Classic” 서비스가 나타나는 것을 확인할 수 있습니다.

<그림 29>와 같이 대시보드에서 Compute Classic의 메뉴 아이콘을 클릭하고 “Compute Classic 서비스 콘솔 열기” 메뉴를 선택합니다.

<그림 30>은 Compute Classic 서비스 콘솔 페이지입니다. Compute Classic 서비스 콘솔 페이지의 왼쪽 위 메뉴 아이콘을 선택하면 현재 접근 가능한 서비스 목록을 확인할 수 있습니다.

<그림 31>과 같이 서비스 목록을 확인한 후, 메뉴에서 “대시보드”를 클릭하여 Oracle Cloud 대시보드 페이지로 이동할 수 있습니다.

<그림 32>와 같이 대시보드에서도 왼쪽 위 메뉴 아이콘을 클릭하여 접근 가능한 서비스 목록을 확인할 수 있습니다.

<그림 33>에서 출력된 목록에서 서비스명을 선택하면, 해당 서버스의 서비스 콘솔로 이동합니다.

요약

Oracle Cloud Trial 프로그램을 등록하고 Oracle Cloud에 로그인하는 전체 과정을 살펴보았습니다. Oracle Cloud Trial 등록에 필요한 전체 시간은 약 15분 정도입니다. 회사 메일이나 Naver/Daum 메일로 오라클 계정을 만들고 Oracle Cloud Trial을 신청할 경우 대부분 15분 정도면 처리가 완료됩니다.

Oracle Cloud의 Compute CS(IaaS)의 1 OCPU/hr 비용은 $0.1입니다. Oracle Cloud Trial의 $300 credit로 완전한 형태의 멀티티어 시스템을 구성하고, 정밀한 테스트 및 검증이 가능합니다.

Oracle Cloud Trial로 Oracle Cloud의 20개 서비스를 사용하실 수 있습니다. Oracle Cloud Trial의 $300 credit은 서비스 사용 시 분 단위로 차감됩니다.
